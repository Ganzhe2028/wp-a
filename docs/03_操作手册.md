# OWeek 个人主页系统 · 操作手册 v1.0

面向：接手开发的同事、coding agent

---

## 1. 环境准备

### 生产地址

线上已部署：`https://oweek26.vercel.app`（Vercel，GitHub `main` 分支 push 自动部署）

### Node.js

需要 Node.js 18 或更新版本。从 [nodejs.org](https://nodejs.org) 下载 LTS 版安装。

验证：

```bash
node -v   # 应 ≥ v18
npm -v    # 应 ≥ v9
```

### 代码获取

```bash
git clone <仓库地址>
cd wp-a
npm install
```

如果 `npm install` 很慢（国内网络），先设淘宝镜像：

```bash
npm config set registry https://registry.npmmirror.com
```

### 数据库：Neon PostgreSQL

1. 注册 [neon.tech](https://neon.tech)（GitHub 登录最快）
2. 创建 project，region 选新加坡 `ap-southeast-1`
3. 拿到两个连接串：
   - **Pooled** 连接串 → `DATABASE_URL`（带 `-pooler` 的 host）
   - **Direct** 连接串 → `DIRECT_URL`

### 图片存储：Cloudflare R2

1. 登录 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧 R2 → Create bucket，名称 `owek-images`，region 选 Asia Pacific
3. 进桶 Settings → Public Access → Enable，记下公开域名
4. Settings → CORS Policy → 添加：

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "PUT"],
    "AllowedHeaders": ["*"],
    "MaxAgeSeconds": 3600
  }
]
```

5. R2 首页右上角 Manage R2 API Tokens → Create Token → Object Read & Write → 记下 Account ID、Access Key ID、Secret Access Key

## 2. 配置

复制环境变量模板并填入实际值：

```bash
cp .env.example .env
```

必须填的变量：

| 变量 | 说明 | 从哪拿 |
|---|---|---|
| `DATABASE_URL` | Neon pooled 连接串 | Neon dashboard → Connection string → Pooled |
| `DIRECT_URL` | Neon direct 连接串 | Neon dashboard → Connection string → Direct |
| `R2_ACCOUNT_ID` | Cloudflare 账号 ID | R2 API Tokens 页面 → URL 里那串 |
| `R2_ACCESS_KEY_ID` | R2 API 密钥 ID | 创建 token 时显示 |
| `R2_SECRET_ACCESS_KEY` | R2 API 密钥 | 创建 token 时显示（只出现一次） |
| `R2_BUCKET` | 桶名 | 填 `owek-images` |
| `R2_PUBLIC_BASE_URL` | 公开域名 | 桶 Settings → Public Access → 域名 |
| `ADMIN_PASSWORD` | 后台口令 | 自己设 |
| `APP_BASE_URL` | 应用根 URL | 本地 `http://localhost:3000`，生产填真实域名 |

## 3. 启动

```bash
npx prisma migrate dev    # 首次：建表
npm run dev               # 启动 → http://localhost:3000
```

验证：

| 检查 | 命令 | 预期 |
|---|---|---|
| 类型 | `npx tsc --noEmit` | 无输出 |
| 构建 | `npm run build` | `✓ Compiled successfully` |
| 数据库 | 访问 `/admin`，用口令登录后导入一条测试数据 | 能成功导入 |

## 4. 页面说明

| 路径 | 用途 | 谁用 |
|---|---|---|
| `/` | 首页 | 入口，跳后台用 |
| `/admin` | 运营后台 | 你们团队 |
| `/u/{code}` | 个人主页 | 新生、浏览者 |
| `/loc/{code}` | 位置页 | 浏览者 |
| `/edit/{token}` | 编辑页 | 新生（每人一个专属链接） |
| `/me/collection` | 我的收藏 | 浏览者（本地存储） |

## 5. API 速查

全部接口：

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| `POST` | `/api/admin/login` | 口令 | `{ password }` → 返回 cookie |
| `POST` | `/api/admin/import` | cookie | `{ rows: [{ chineseName, englishName?, grade? }] }` → 返回 code + editToken |
| `POST` | `/api/admin/location` | cookie | `{ code, name, room, seat, grade? }` → 更新位置页 |
| `POST` | `/api/admin/takedown` | cookie | `{ type, id, hidden }` → 下架/恢复 |
| `GET` | `/api/admin/export` | cookie | 导出 CSV（每人 5 列） |
| `GET` | `/api/admin/persons` | cookie | 所有人 JSON（下架面板用） |
| `GET` | `/api/admin/settings` | cookie | 所有系统设置键值对 |
| `PATCH` | `/api/admin/settings` | cookie | 更新单个系统设置 |
| `GET` | `/api/me?token=` | token | 获取自己的数据 |
| `PATCH` | `/api/me?token=` | token | 更新自己的数据（bio ≤ 80、无头像禁发布） |
| `POST` | `/api/upload-url?token=` | token | `{ contentType }` → presigned PUT URL + public URL |
| `POST` | `/api/me/images?token=` | token | 保存图片记录 |
| `DELETE` | `/api/me/images?token=&id=` | token | 删除图片（R2 + DB） |

## 6. 运营流程

### 导入学生

1. 打开 `/admin`，用口令登录
2. 切到「导入名单」tab
3. 粘贴数据，格式：`englishName,chineseName,grade`（每行一人）
4. 点「解析」预览 → 「导入」写库
5. 每人自动生成 code（短码）和 editToken（编辑链接）

### 填位置页

1. 切到「位置编辑」tab
2. 输入短码，填房间号 + 座位号
3. 保存

### 导出链接表

1. 切到「导出」tab
2. 点「导出 CSV」→ 拿到所有人的主页链接、位置链接、编辑链接
3. 用这些链接写 NFC 标签、印二维码、发编辑链接给学生

### 下架

1. 切到「下架」tab
2. 搜索要下架的人
3. 切换隐藏开关

## 7. 常见问题

### 数据库迁移失败

```bash
npx prisma migrate dev --name fix
```

如果提示 schema 不一致，先 `npx prisma db push` 强制对齐。

### 图片上传失败

1. 确认 `.env` 里 R2 五个变量都已填
2. 确认 CORS 策略已添加
3. 检查 R2 桶是否开启了 Public Access

### 构建报错

```bash
npx tsc --noEmit    # 先看类型错误
npm run build        # 再看构建错误
```

### Windows 下 Prisma 报错

如果 `prisma migrate dev` 报引擎下载失败，设环境变量后重试：

```powershell
$env:PRISMA_ENGINES_MIRROR="https://registry.npmmirror.com/-/binary/prisma/"
npx prisma migrate dev
```
