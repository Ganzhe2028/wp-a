# OWeek 个人主页系统 · 操作手册 v2.0

面向：接手开发的同事、coding agent

---

## 1. 环境准备

### 生产地址

线上已部署：`https://msoweek.site`（Vercel，GitHub `main` 分支 push 自动部署）

### Node.js

需要 Node.js 18 或更新版本。从 [nodejs.org](https://nodejs.org) 下载 LTS 版安装。

验证：

```bash
node -v   # 应 ≥ v18
npm -v    # 应 ≥ v9
```

### 代码获取

```bash
git clone <仓库地址>
cd wp-a
npm install
```

如果 `npm install` 很慢（国内网络），先设淘宝镜像：

```bash
npm config set registry https://registry.npmmirror.com
```

### 数据库：Neon PostgreSQL

1. 注册 [neon.tech](https://neon.tech)（GitHub 登录最快）
2. 创建 project，region 选新加坡 `ap-southeast-1`
3. 拿到两个连接串：
   - **Pooled** 连接串 → `DATABASE_URL`（带 `-pooler` 的 host）
   - **Direct** 连接串 → `DIRECT_URL`

### 图片存储：Cloudflare R2

1. 登录 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧 R2 → Create bucket，名称 `owek-images`，region 选 Asia Pacific
3. 进桶 Settings → Public Access → Enable，记下公开域名
4. Settings → CORS Policy → 添加：

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "PUT"],
    "AllowedHeaders": ["*"],
    "MaxAgeSeconds": 3600
  }
]
```

5. R2 首页右上角 Manage R2 API Tokens → Create Token → Object Read & Write → 记下 Account ID、Access Key ID、Secret Access Key

## 2. 配置

复制环境变量模板并填入实际值：

```bash
cp .env.example .env
```

必须填的变量：

| 变量 | 说明 | 从哪拿 |
|---|---|---|
| `DATABASE_URL` | Neon pooled 连接串 | Neon dashboard → Connection string → Pooled |
| `DIRECT_URL` | Neon direct 连接串 | Neon dashboard → Connection string → Direct |
| `R2_ACCOUNT_ID` | Cloudflare 账号 ID | R2 API Tokens 页面 → URL 里那串 |
| `R2_ACCESS_KEY_ID` | R2 API 密钥 ID | 创建 token 时显示 |
| `R2_SECRET_ACCESS_KEY` | R2 API 密钥 | 创建 token 时显示（只出现一次） |
| `R2_BUCKET` | 桶名 | 填 `owek-images` |
| `R2_PUBLIC_BASE_URL` | 公开域名 | 桶 Settings → Public Access → 域名 |
| `ADMIN_PASSWORD` | 后台口令 | 自己设 |
| `SESSION_SECRET` | 学生 session 签名密钥 | `openssl rand -base64 48` 生成，≥32 字节 |
| `APP_BASE_URL` | 应用根 URL | 本地 `http://localhost:3000`，生产填真实域名 |

## 5. API 速查

全部接口：

| 方法 | 路径 | 鉴权 | 说明 |
|---|---|---|---|
| `POST` | `/api/auth/login` | 无 | `{ username, password }` → 返回 session cookie |
| `POST` | `/api/auth/logout` | cookie | 清 session cookie |
| `GET` | `/api/me` | session | 获取自己的数据 |
| `PATCH` | `/api/me` | session | 更新自己的数据（bio ≤ 80） |
| `POST` | `/api/upload-url` | session | `{ contentType }` → presigned PUT URL + public URL |
| `POST` | `/api/me/images` | session | 保存图片记录 |
| `DELETE` | `/api/me/images?id=` | session | 删除图片（R2 + DB） |
| `POST` | `/api/favorites` | session | `{ code }` → toggle 收藏 |
| `GET` | `/api/me/favorites` | session | 我收藏的人列表 |
| `POST` | `/api/admin/login` | 口令 | `{ password }` → 返回 admin cookie |
| `GET` | `/api/admin/session` | admin cookie | `{ authed: true/false }` → 校验当前会话 |
| `POST` | `/api/admin/logout` | admin cookie | 清除 admin cookie 登出 |
| `POST` | `/api/admin/import` | admin cookie | `{ rows: [{ chineseName, englishName?, grade?, username? }] }` → 返回含明文密码 |
| `POST` | `/api/admin/location` | admin cookie | `{ code, name, room, seat, grade? }` → 更新位置页 |
| `POST` | `/api/admin/takedown` | admin cookie | `{ type, id, hidden }` → 下架/恢复 |
| `GET` | `/api/admin/export` | admin cookie | 导出 CSV（chineseName, englishName, username, homepage, location） |
| `POST` | `/api/admin/reset-password` | admin cookie | `{ code }` → 返回新明文密码（只出现一次） |
| `GET` | `/api/admin/persons` | admin cookie | 所有人 JSON |
| `GET` | `/api/admin/settings` | admin cookie | 所有系统设置键值对 |
| `PATCH` | `/api/admin/settings` | admin cookie | 更新单个系统设置 |
| `GET` | `/api/admin/qr/print` | admin cookie | 批量生成 QR 码打印页 |
| `GET` | `/api/settings?key=` | 无 | 公开读单个系统设置 |

## 6. 运营流程

### 导入学生

1. 打开 `/admin`，用口令登录
2. 切到「导入名单」tab
3. 粘贴数据，格式：`englishName,chineseName,grade`（每行一人）
4. 点「解析」预览 → 「导入」写库
5. 每人自动生成：code（短码）、username（默认同 code）、密码（12 位可打印字符，含大小写字母、数字和少量安全符号）
6. **导入成功后立即点「下载帐密 CSV」**——明文密码只在这一次出现
7. 把 CSV 里的用户名和密码印在纸条上发给学生

### 填位置页

1. 切到「位置编辑」tab
2. 输入短码，填房间号 + 座位号
3. 保存

### 导出链接表

1. 切到「导出」tab
2. 点「下载 CSV」→ 拿到所有人的中文名、英文名、用户名、主页链接、展位链接
3. **密码不可导出**——存库的是 hash，导出 CSV 不含密码
4. 用链接写 NFC 标签、印二维码

### 下架

1. 切到「下架」tab
2. 搜索要下架的人
3. 切换隐藏开关

### 系统设置

当前「系统设置」tab 暂无运营可配开关。主页可见性由系统自动判断：学生上传头像并保存资料后，主页即变为可见；运营仍可通过「下架」tab 隐藏特定页面。

### 生成 QR 码

1. 切到「QR 码」tab
2. 预览全部人员列表（姓名、短码、发布状态）
3. 点「打开打印页」→ 新窗口自动弹出打印对话框
4. 每人生成 2 张 QR 码：主页 QR（`/u/码`）和位置页 QR（`/loc/码`），4 列 A4 网格布局
5. 建议用 A4 纸打印后裁剪，贴到明信片和展板上

## 7. 账号管理（v2.0）

### 学生登录

学生在首页输入用户名和密码登录。用户名默认是短码（code）。登录后系统下发 session cookie（14 天有效），活动期间无需重复登录。

### 忘记密码

学生没有自助重置入口。流程：组长联系开发 → 开发在 `/admin` →「重置密码」tab 输入短码 → 系统返回新明文密码（只显示一次）。

### 登录态与浏览器

- Session cookie 是服务端 httpOnly 下发的，不受 iOS Safari ITP 影响
- **不能在微信内置浏览器里登录**——微信和 Safari 是独立 cookie jar
- 登录页会检测微信 UA，若在微信内打开，会提示「请在 Safari/系统默认浏览器中打开」
- 全程统一用系统默认浏览器

### 系统设置

见第 6 节「系统设置」。当前暂无学生可见性开关，主页自动发布。

## 8. 常见问题

### 数据库迁移失败

```bash
npx prisma migrate dev --name fix
```

如果提示 schema 不一致，先 `npx prisma db push` 强制对齐。

### 学生无法登录

1. 确认 Vercel 环境变量里 `SESSION_SECRET` 已设置且够长（≥32 字节）
2. 确认账号已导入且明文密码已下载（存库的是 hash，无法还原）
3. 密码丢失直接用「重置密码」功能生成新密码

### 图片上传失败

1. 确认 `.env` 里 R2 五个变量都已填
2. 确认 CORS 策略已添加
3. 检查 R2 桶是否开启了 Public Access

### 构建报错

```bash
npx tsc --noEmit    # 先看类型错误
npm run build        # 再看构建错误
```

### Windows 下 Prisma 报错

如果 `prisma migrate dev` 报引擎下载失败，设环境变量后重试：

```powershell
$env:PRISMA_ENGINES_MIRROR="https://registry.npmmirror.com/-/binary/prisma/"
npx prisma migrate dev
```
